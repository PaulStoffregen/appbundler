AppBundler

Description

Generates a native launcher for a Java application.

Parameters

Attribute Description Required
outputdirectory The output folder for the generated bundle. Yes
name The name of the generated bundle. Corresponds to the CFBundleName key in the Info.plist file. Yes
displayname The display name of the generated bundle. Corresponds to the CFBundleDisplayName key in the Info.plist file. Yes
identifier An identifier string that specifies the application type of the bundle. Corresponds to the CFBundleIdentifier key in the Info.plist file. Yes
icon The icon file for the bundle. Corresponds to the CFBundleIconFile key in the Info.plist file. No
shortversion The release version number string for the bundle. Corresponds to the CFBundleShortVersionString key in the Info.plist file. No
signature The four-letter code identifying the bundle creator. Corresponds to the CFBundleSignature key in the Info.plist file. No
copyright The copyright notice for the bundle. Corresponds to the NSHumanReadableCopyright key in the Info.plist file. No
runtime The Java runtime to embed in the bundle. Corresponds to the JVMRuntime key in the Info.plist file. If specified, the contents of this directory will be copied to the Contents/PlugIns/ folder of the generated bundle. No; if unspecified, the target system must have a shared JRE installed in /Library/Internet Plug-Ins/JavaAppletPlugin.plugin.
mainclassname The name of the bundled application's main class. Corresponds to the JVMMainClassName key in the Info.plist file. Yes
classpath The classpath of the bundled application. If an entry refers to a directory, it will be recursively copied to the Contents/Java/Classes/ folder of the generated bundle. Otherwise, the entryt will be copied to the Contents/Java/ folder of the generated bundle. Yes, unless nested <classpath> elements are present.
nativelibraries Any native libraries required by the bundled application. Entries will be copied to the Contents/Java/ folder of the generated bundle. Yes, unless nested <nativelibraries> elements are present.

Parameters specified as nested elements

classpath

The classpath and nativelibraries attributes are path-like structures that can also be set via nested <classpath> and <nativelibraries> elements, respectively.

option

Specifies a command-line option to be passed to the JVM at startup.

Attribute Description Required
value The option value. Yes

argument

Specifies a command-line argument to be passed to the application at startup.

Attribute Description Required
value The argument value. Yes

Examples

Generate a launcher for the "Swing Set" demo, bundling the JRE defined by the JAVA_HOME environment variable with the resulting executable:

<-- Import environment variables -->
<property environment="env"/>

<-- Define the appbundler task -->
<taskdef name="bundleapp" classname="com.oracle.appbundler.AppBundlerTask"/>

<-- Create the app bundle -->
<target name="bundle-swingset" depends="package">
    <bundleapp outputdirectory="."
        name="SwingSet2"
        displayname="SwingSet 2"
        identifier="com.oracle.javax.swing.SwingSet2"
        shortversion="1.0"
        runtime="${env.JAVA_HOME}/../.."
        mainclassname="SwingSet2">
        <classpath file="/Library/Java/Demos/JFC/SwingSet2/SwingSet2.jar"/>
        <option value="-Dapple.laf.useScreenMenuBar=true"/>
    </bundleapp>
</target>
